## 1. 系统概述

本系统是面向连锁酒店的互联网管理计费平台，支持以下核心功能：

- **会员计费**：包天 / 包月 / 包年 / 按 MB/GB 流量计费
- **设备管控**：对接 MikroTik 路由器，支持 RADIUS 标准协议设备
- **实时扣费**：上网时按流量实时从余额扣除，余额不足自动断网
- **充值卡**：批量生成充值卡，住客自助核销
- **多酒店管理**：一个集团统一管理多家酒店，数据互相隔离

### 角色说明

| 角色 | 权限 |
|------|------|
| **SUPER_ADMIN** | 超级管理员：管理所有租户、所有酒店 |
| **ADMIN** | 管理员：管理本租户下的酒店、会员、套餐、设备 |
| **OPERATOR** | 操作员：会员管理、充值、计费查询 |
| **VIEWER** | 只读：查看报表和数据 |

## 2. 快速入门

### 2.1 部署方式

#### Docker 一键部署（推荐）

```bash
cd hotel-wifi-platform
docker-compose up -d
```

首次启动会自动创建数据库和表结构，并插入默认数据。

#### 手动部署

**前置条件**：Java 17+ / MySQL 8.0 / Redis 7.x

```bash
# 1. 创建数据库
mysql -u root -p -e "CREATE DATABASE hotel_wifi DEFAULT CHARSET utf8mb4"

# 2. 修改配置文件
# 编辑 hotel-wifi-api/src/main/resources/application.yml
# 修改数据库连接地址、用户名、密码

# 3. 启动
mvn clean package -DskipTests -pl hotel-wifi-api -am
java -jar hotel-wifi-api/target/hotel-wifi-api-1.0.0-SNAPSHOT.jar
```

### 2.2 访问系统

| 入口 | 地址 | 说明 |
|------|------|------|
| 管理后台 | `http://服务器IP` | 管理员操作界面 |
| API 文档 | `http://服务器IP:8080/doc.html` | 开发调试用 |

### 2.3 默认账号

| 项目 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `admin123` |
| 租户ID | `1` |

> ⚠️ **首次登录后请立即修改密码！**

## 5. 租户管理

> 仅 SUPER_ADMIN 可见

### 5.1 什么是租户？
租户代表一个**酒店管理集团**，一个租户下可管理多家酒店。

### 5.2 新增租户

1. 点击 **新增租户** 按钮
2. 填写以下信息：
   - **租户名称**（必填）：如"XX酒店管理集团"
   - **联系人**：集团负责人姓名
   - **联系电话**：手机或座机
   - **邮箱**：联系邮箱
   - **地址**：办公地址
3. 点击 **保存**

### 5.3 编辑租户

点击租户列表右侧的 **编辑** 按钮，修改信息后保存。

### 5.4 停用/启用

点击 **停用** 可暂停该租户下的所有业务；点击 **启用** 恢复。

> ⚠️ 停用租户后，该租户下所有酒店和会员将无法正常使用。

## 6. 酒店管理

### 6.1 新增酒店

1. 点击 **新增酒店**
2. 填写信息：
   - **酒店名称**（必填）：如"XX大酒店"
   - **酒店编码**：内部编号，如 `BJ001`
   - **房间数**：酒店实际客房数量
   - **最大在线**：同时上网的最大用户数（建议设为房间数 × 2）
   - **地址**：酒店详细地址
   - **联系电话**：前台电话
3. 点击 **保存**

### 6.2 编辑酒店

点击列表中的 **编辑**，修改后保存。

### 6.3 停用/启用

停用后该酒店会员将无法认证上网。

## 7. 会员管理

### 7.1 会员列表

显示该租户下所有会员，信息包括：
- 用户名、姓名、手机号
- 账户余额（元）
- 套餐到期时间
- 状态（正常/停用/黑名单）

### 7.2 新增会员

1. 点击 **新增会员**
2. 填写信息：
   - **用户名**（必填）：会员登录账号
   - **密码**（必填）：初始密码，会员可自行修改
   - **姓名**：真实姓名
   - **手机号**：用于短信认证和到期提醒
3. 点击 **保存**

### 7.3 编辑会员

1. 点击 **编辑**
2. 修改姓名、手机号等信息（用户名不可修改）
3. 点击 **保存**

### 7.4 会员充值

1. 点击会员列表中的 **充值** 按钮
2. 输入充值金额（元），如 `100`
3. 点击 **确认充值**

充值后金额立即到账，会员可使用余额购买套餐。

### 7.5 停用/启用

- **停用**：暂停会员使用权限
- **启用**：恢复正常
- 状态为 **黑名单** 的会员将被永久禁止使用

## 8. 套餐管理

### 8.1 套餐类型说明

| 类型 | 扣费方式 | 适用场景 |
|------|---------|---------|
| **包时** | 一次性扣费，到期自动断网 | 住客住宿期间上网 |
| **包流量** | 按实际使用流量实时扣费 | 临时上网、会议 |
| **混合** | 先扣时长费，超额部分按流量扣 | 灵活计费 |

### 8.2 新增套餐

1. 点击 **新增套餐**
2. 选择 **计费类型**
3. 配置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 套餐名称 | 显示名称 | "24小时上网套餐" |
| 时长(秒) | 包时套餐的有效时长 | 86400 = 1天 |
| 流量(字节) | 包流量套餐的流量配额 | 1073741824 = 1GB |
| 价格(元) | 售价 | 9.90 |
| 最大设备数 | 同一账号同时在线设备数 | 2 |
| 下行限速(bps) | 下载速度限制 | 10485760 = 10Mbps |
| 上行限速(bps) | 上传速度限制 | 5242880 = 5Mbps |

4. 点击 **保存**

### 8.3 常用参数换算

| 时长 | 秒数 |
|------|------|
| 1天 | 86400 |
| 7天 | 604800 |
| 30天 | 2592000 |
| 365天 | 31536000 |

| 流量 | 字节数 |
|------|--------|
| 500MB | 524288000 |
| 1GB | 1073741824 |
| 5GB | 5368709120 |
| 10GB | 10737418240 |

| 网速 | bps |
|------|-----|
| 1Mbps | 1048576 |
| 5Mbps | 5242880 |
| 10Mbps | 10485760 |
| 50Mbps | 52428800 |
| 100Mbps | 104857600 |

### 8.4 启用/停用套餐

停用的套餐不会在前端 Portal 页面显示，已有用户不受影响。

## 10. 充值卡管理

### 10.1 什么是充值卡？

充值卡是一组预生成的卡号和密码，住客可通过卡号密码自助充值，无需前台操作。

### 10.2 批量生成充值卡

1. 点击 **批量生成**
2. 设置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 生成数量 | 本次生成多少张卡 | 100 |
| 面值(元) | 每张卡的面值 | 10 |
| 关联套餐ID | 填入套餐ID=套餐卡；不填/0=余额卡 | 空 |
| 过期时间 | 超过此时间卡失效 | 2026-12-31 |

3. 点击 **生成**
4. 弹出结果窗口，显示所有卡号和密码
5. 点击 **全部复制** 导出，或截图保存

### 10.3 充值卡分类

| 类型 | 说明 | 核销效果 |
|------|------|---------|
| **余额卡** | 不关联套餐 | 充值金额到会员余额 |
| **套餐卡** | 关联指定套餐ID | 激活对应套餐 + 充值等额余额 |

### 10.4 核销充值卡

住客拿到充值卡后，管理员可以帮其核销：

1. 点击 **核销充值卡**
2. 输入 **卡号**、**密码**、**会员ID**
3. 点击 **核销**

核销成功后金额立即到账。

### 10.5 作废充值卡

对于未使用的充值卡，可点击 **作废** 使其失效。

## 11. 设备管理

### 11.1 添加路由器

1. 点击 **添加设备**
2. 填写配置：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 设备名称 | 自定义标识 | - |
| IP/域名 | 路由器的管理IP | - |
| API端口 | REST API 端口 | 8728 |
| API用户名 | MikroTik 管理员账号 | admin |
| API密码 | MikroTik 管理员密码 | - |
| 所属酒店ID | 关联的酒店 | - |
| Hotspot Server | MikroTik Hotspot 服务名 | - |
| IP地址池 | 分配给用户的IP段 | - |

3. 点击 **保存**

### 11.2 测试连接

1. 点击设备列表中的 **测试**
2. 系统会尝试通过 REST API 连接路由器
   - 🟢 **连接成功**：设备正常在线
   - 🔴 **连接失败**：检查 IP、端口、用户名、密码是否正确

### 11.3 同步配置

1. 点击 **同步**
2. 系统会将套餐限速规则等配置下发到路由器
3. 成功后显示"配置同步完成"

### 11.4 设备状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| 在线 | 🟢 绿色 | 设备正常，心跳通过 |
| 离线 | ⚪ 灰色 | 设备不可达 |
| 异常 | 🔴 红色 | 通信异常 |
| 维护 | 🟡 黄色 | 维护中 |

### 11.5 支持的设备类型

| 设备 | 接口方式 | 说明 |
|------|---------|------|
| MikroTik RouterOS | REST API | 端口 8728/443，自动轮询流量 |
| 华为 ME60/MA5800 | RADIUS | 端口 1812/1813 |
| H3C SR88/CR16000 | RADIUS | 端口 1812/1813 |
| 思科 ASR1000 | RADIUS | 端口 1812/1813 |
| Juniper MX | RADIUS | 端口 1812/1813 |

> RADIUS 模式下，设备列表中的设备类型需选择非 `MIKROTIK` 的其他选项。

## 13. 常见问题

### Q1: 忘记管理员密码怎么办？

连接数据库，执行以下 SQL 重置密码为 `admin123`：

```sql
UPDATE sys_user SET password = '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iKTVKIUi'
WHERE username = 'admin';
```

### Q2: 前端页面无法加载？

1. 确认后端 API 服务已启动：访问 `http://服务器IP:8080/doc.html`
2. 检查 Nginx 配置是否指向正确的前端和后端地址
3. 清除浏览器缓存后重试

### Q3: 会员反映无法上网？

按以下步骤排查：
1. **查状态**：会员管理 → 确认该会员状态为"正常"
2. **查余额**：确认余额充足或套餐未过期
3. **查设备**：设备管理 → 确认路由器状态为"在线"
4. **测试连接**：设备管理 → 对目标路由器点击"测试"

### Q4: 扣费不准确？

系统采用 30 秒轮询 MikroTik 获取流量数据，60 秒结算一次。扣费存在 1-2 分钟延迟属于正常现象。如需更高精度，可在 `application.yml` 中调整：

```yaml
billing:
  poll-interval-seconds: 15   # 缩小轮询间隔
  deduct-interval-seconds: 30 # 缩小扣费间隔
```

### Q5: 如何修改系统 Logo 和品牌名称？

前端品牌信息在以下文件中：
- `hotel-wifi-frontend/src/components/layout/AppLayout.vue` — 侧边栏 Logo
- `hotel-wifi-frontend/src/views/login/LoginView.vue` — 登录页标题
- `hotel-wifi-frontend/index.html` — 浏览器标签页标题

### Q6: 数据库备份建议

```bash
# 每天凌晨 2 点自动备份
0 2 * * * /usr/bin/mysqldump -u root -p密码 hotel_wifi | gzip > /backup/hotel_wifi_$(date +\%Y\%m\%d).sql.gz

# 保留最近 30 天的备份
find /backup -name "hotel_wifi_*.sql.gz" -mtime +30 -delete
```

### Q7: RADIUS 设备配置要点？

1. 在 NAS 设备上配置 RADIUS 服务器地址为本系统 IP
2. 认证端口：**1812**，计费端口：**1813**
3. 共享密钥（Secret）需要在设备表 `router_device.api_password` 中配置
4. 设备类型选择 `HUAWEI` / `H3C` / `CISCO` / `JUNIPER`（非 MIKROTIK）

| 指标 | 单实例 | 集群 |
