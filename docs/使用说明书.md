## 1. 系统概述

本系统是面向连锁酒店的互联网管理计费平台，支持以下核心功能：

- **会员计费**：包天 / 包月 / 包年 / 按 MB/GB 流量计费
- **设备管控**：对接 MikroTik 路由器，支持 RADIUS 标准协议设备
- **实时扣费**：上网时按流量实时从余额扣除，余额不足自动断网
- **充值卡**：批量生成充值卡，住客自助核销
- **多酒店管理**：一个集团统一管理多家酒店，数据互相隔离

### 角色说明

| 角色 | 权限 |
|------|------|
| **SUPER_ADMIN** | 超级管理员：管理所有租户、所有酒店 |
| **ADMIN** | 管理员：管理本租户下的酒店、会员、套餐、设备 |
| **OPERATOR** | 操作员：会员管理、充值、计费查询 |
| **VIEWER** | 只读：查看报表和数据 |

## 2. 快速入门

### 2.1 部署方式

#### Docker 一键部署（推荐）

```bash
cd hotel-wifi-platform
docker-compose up -d
```

首次启动会自动创建数据库和表结构，并插入默认数据。

#### 手动部署

**前置条件**：Java 17+ / MySQL 8.0 / Redis 7.x

```bash
# 1. 创建数据库
mysql -u root -p -e "CREATE DATABASE hotel_wifi DEFAULT CHARSET utf8mb4"

# 2. 修改配置文件
# 编辑 hotel-wifi-api/src/main/resources/application.yml
# 修改数据库连接地址、用户名、密码

# 3. 启动
mvn clean package -DskipTests -pl hotel-wifi-api -am
java -jar hotel-wifi-api/target/hotel-wifi-api-1.0.0-SNAPSHOT.jar
```

### 2.2 访问系统

| 入口 | 地址 | 说明 |
|------|------|------|
| 管理后台 | `http://服务器IP` | 管理员操作界面 |
| API 文档 | `http://服务器IP:8080/doc.html` | 开发调试用 |

### 2.3 默认账号

| 项目 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `admin123` |
| 租户ID | `1` |

> ⚠️ **首次登录后请立即修改密码！**

## 5. 租户管理

> 仅 SUPER_ADMIN 可见

### 5.1 什么是租户？
租户代表一个**酒店管理集团**，一个租户下可管理多家酒店。

### 5.2 新增租户

1. 点击 **新增租户** 按钮
2. 填写以下信息：
   - **租户名称**（必填）：如"XX酒店管理集团"
   - **联系人**：集团负责人姓名
   - **联系电话**：手机或座机
   - **邮箱**：联系邮箱
   - **地址**：办公地址
3. 点击 **保存**

### 5.3 编辑租户

点击租户列表右侧的 **编辑** 按钮，修改信息后保存。

### 5.4 停用/启用

点击 **停用** 可暂停该租户下的所有业务；点击 **启用** 恢复。

> ⚠️ 停用租户后，该租户下所有酒店和会员将无法正常使用。

## 6. 酒店管理

### 6.1 新增酒店

1. 点击 **新增酒店**
2. 填写信息：
   - **酒店名称**（必填）：如"XX大酒店"
   - **酒店编码**：内部编号，如 `BJ001`
   - **房间数**：酒店实际客房数量

   - **地址**：酒店详细地址
   - **联系电话**：前台电话
3. 点击 **保存**

### 6.2 编辑酒店

点击列表中的 **编辑**，修改后保存。

### 6.3 停用/启用

停用后该酒店会员将无法认证上网。

## 7. 会员管理

### 7.1 会员列表

显示该租户下所有会员，信息包括：
- 用户名、姓名、手机号
- 账户余额（元）
- 套餐到期时间
- 状态（正常/停用/黑名单）

### 7.2 新增会员

1. 点击 **新增会员**
2. 填写信息：
   - **用户名**（必填）：会员登录账号
   - **密码**（必填）：初始密码，会员可自行修改
   - **姓名**：真实姓名
   - **手机号**：用于短信认证和到期提醒
3. 点击 **保存**

### 7.3 编辑会员

1. 点击 **编辑**
2. 修改姓名、手机号等信息（用户名不可修改）
3. 点击 **保存**

### 7.4 会员充值

1. 点击会员列表中的 **充值** 按钮
2. 输入充值金额（元），如 `100`
3. 点击 **确认充值**

充值后金额立即到账，会员可使用余额购买套餐。

### 7.5 停用/启用

- **停用**：暂停会员使用权限
- **启用**：恢复正常
- 状态为 **黑名单** 的会员将被永久禁止使用

## 8. 套餐管理

### 8.1 套餐类型说明

| 类型 | 扣费方式 | 适用场景 |
|------|---------|---------|
| **包时** | 一次性扣费，到期自动断网 | 住客住宿期间上网 |
| **包流量** | 按实际使用流量实时扣费 | 临时上网、会议 |
| **混合** | 先扣时长费，超额部分按流量扣 | 灵活计费 |

### 8.2 新增套餐

1. 点击 **新增套餐**
2. 选择 **计费类型**
3. 配置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 套餐名称 | 显示名称 | "24小时上网套餐" |
| 时长(秒) | 包时套餐的有效时长 | 86400 = 1天 |
| 流量(字节) | 包流量套餐的流量配额 | 1073741824 = 1GB |
| 价格(元) | 售价 | 9.90 |
| 最大设备数 | 同一账号同时在线设备数 | 2 |
| 下行限速(bps) | 下载速度限制 | 10485760 = 10Mbps |
| 上行限速(bps) | 上传速度限制 | 5242880 = 5Mbps |

4. 点击 **保存**

### 8.3 常用参数换算

| 时长 | 秒数 |
|------|------|
| 1天 | 86400 |
| 7天 | 604800 |
| 30天 | 2592000 |
| 365天 | 31536000 |

| 流量 | 字节数 |
|------|--------|
| 500MB | 524288000 |
| 1GB | 1073741824 |
| 5GB | 5368709120 |
| 10GB | 10737418240 |

| 网速 | bps |
|------|-----|
| 1Mbps | 1048576 |
| 5Mbps | 5242880 |
| 10Mbps | 10485760 |
| 50Mbps | 52428800 |
| 100Mbps | 104857600 |

### 8.4 启用/停用套餐

停用的套餐不会在前端 Portal 页面显示，已有用户不受影响。

## 10. 充值卡管理

### 10.1 什么是充值卡？

充值卡是一组预生成的卡号和密码，住客可通过卡号密码自助充值，无需前台操作。

### 10.2 批量生成充值卡

1. 点击 **批量生成**
2. 设置参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 生成数量 | 本次生成多少张卡 | 100 |
| 面值(元) | 每张卡的面值 | 10 |
| 关联套餐ID | 填入套餐ID=套餐卡；不填/0=余额卡 | 空 |
| 过期时间 | 超过此时间卡失效 | 2026-12-31 |

3. 点击 **生成**
4. 弹出结果窗口，显示所有卡号和密码
5. 点击 **全部复制** 导出，或截图保存

### 10.3 充值卡分类

| 类型 | 说明 | 核销效果 |
|------|------|---------|
| **余额卡** | 不关联套餐 | 充值金额到会员余额 |
| **套餐卡** | 关联指定套餐ID | 激活对应套餐 + 充值等额余额 |

### 10.4 核销充值卡

住客拿到充值卡后，管理员可以帮其核销：

1. 点击 **核销充值卡**
2. 输入 **卡号**、**密码**、**会员ID**
3. 点击 **核销**

核销成功后金额立即到账。

### 10.5 作废充值卡

对于未使用的充值卡，可点击 **作废** 使其失效。

## 11. 设备管理

设备管理是整个系统的核心模块，负责对接酒店的路由器，实现流量采集、用户认证、实时扣费和强制断网。系统支持两种连接模式：**公网直连**和**Agent 代理穿透**。

---

### 11.1 功能概览

设备管理页面提供以下操作：

| 按钮 | 功能 | 说明 |
|------|------|------|
| **添加设备** | 录入路由器 | 填写连接参数并保存 |
| **编辑** | 修改配置 | 修改 IP/端口/密码等信息 |
| **测试** | 连接测试 | 尝试调用路由器 REST API 验证连通性 |
| **同步** | 配置下发 | 将套餐限速规则等同步到路由器 |
| **Agent** | 检查代理 | 查看内网 Agent 是否在线 |
| **删除** | 移除设备 | 删除该设备记录 |

---

### 11.2 两种连接模式

#### 模式 A：公网直连（适用有公网 IP 或端口映射）

```
 公网服务器 ──REST API──> MikroTik 路由器 (公网可达)
```

**适用场景：** 路由器有公网 IP，或已做端口映射（将 8728 或 443 端口映射到公网）。

**添加步骤：**

1. 登录管理后台 → **设备管理** → 点击 **添加设备**
2. 按下方说明填写表单（详见 11.3 节）
3. 在 `IP/域名` 字段中填写路由器的公网 IP 地址
4. 点击保存后，**点击测试按钮**验证连通性

> **需要路由器端操作：** 在 RouterOS 中执行：
> ```
> /ip service enable www-ssl
> /ip service set www-ssl port=443 disabled=no
> ```
> 然后在防火墙中放行相应端口。

---

#### 模式 B：Agent 代理模式（路由器在内网/NAT 后）⭐ 推荐

```
 公网服务器  <──WebSocket──>  内网 Agent (Linux盒子/RPi)  ──REST API──>  MikroTik
```

**适用场景：** 酒店路由器位于内网（192.168.x.x），无公网 IP，无端口映射。

**工作原理：**

```
  ┌──────────────────────────────────────────────────┐
  │                   公网服务器                        │
  │  /ws/device  (WebSocket 端点，无需 JWT 认证)       │
  │       ↕                                          │
  │  DeviceRelayHandler                               │
  │    └─ AGENTS Map<deviceId, WebSocketSession>      │
  │    └─ PENDING Map<requestId, CompletableFuture>   │
  └────────────────────┬─────────────────────────────┘
                       │ WebSocket (长连接)
  ┌────────────────────┴─────────────────────────────┐
  │              酒店内网 (Linux 盒子)                  │
  │                                                   │
  │  DeviceConnector.jar  (独立 Java 应用)              │
  │    ├─ 注册: {type:"register", deviceId, secret}    │
  │    ├─ 心跳: 每 30s 发送 {type:"heartbeat"}        │
  │    ├─ 扫描: 每 5m 扫描局域网 ARP/DHCP             │
  │    ├─ 采集: 每 60s 采集路由器 CPU/内存/流量        │
  │    └─ 命令: 接收远程命令并代理执行                  │
  │         ├─ mikrotik_get / mikrotik_post            │
  │         ├─ mikrotik_hotspot_active                 │
  │         ├─ scan_network / disconnect_user          │
  │         └─ ping                                   │
  └────────────────────┬─────────────────────────────┘
                       │ OkHttp (本地调用)
  ┌────────────────────┴─────────────────────────────┐
  │              MikroTik 路由器                       │
  │            192.168.1.1:8728                       │
  └──────────────────────────────────────────────────┘
```

**添加步骤：**

**阶段一：录入设备**

1. 登录管理后台 → **设备管理** → 点击 **添加设备**
2. `设备名称`：填写 "某某酒店主路由"
3. `IP/域名`：填写内网 Agent 的标识符，格式为 `ws://` 开头的字符串（如 `ws://agent-hotel-01`），系统据此识别为 Agent 模式
4. `API 端口`、`API 用户名`、`API 密码`：填写 Agent 所在局域网内路由器的实际连接信息
5. 点击保存。此时设备状态显示为 **离线**（Agent 尚未连接）

**阶段二：部署内网 Agent**

在内网任意 Linux 设备（树莓派/工控机/旧 PC/虚拟机）上部署 Agent：

```bash
# 1. 安装 Java 17+
yum install -y java-17-openjdk-headless     # CentOS/RHEL
# 或
apt install -y openjdk-17-jre-headless       # Ubuntu/Debian

# 2. 构建 Agent JAR（在开发机上执行，或使用预编译版本）
cd hotel-wifi-platform/device-connector
mvn clean package -DskipTests

# 3. 上传到内网 Linux 盒子
scp target/device-connector-1.0.0-SNAPSHOT.jar root@<盒子IP>:/opt/hotel-connector/

# 4. 启动 Agent
java -Xmx128m -Xms64m \
  -Dserver=ws://185.239.71.210:8080/ws/device \
  -DdeviceId=1 \
  -Dsecret=设备密钥 \
  -DmikrotikHost=192.168.1.1 \
  -DmikrotikPort=8728 \
  -DmikrotikUser=hotelapi \
  -DmikrotikPassword=你的路由器密码 \
  -jar /opt/hotel-connector/device-connector-1.0.0-SNAPSHOT.jar
```

**启动参数说明：**

| 参数 | 必需 | 说明 | 示例 |
|------|:--:|------|------|
| `-Dserver` | ✅ | 服务器 WebSocket 地址 | `ws://185.239.71.210:8080/ws/device` |
| `-DdeviceId` | ✅ | 与后台录入的**设备 ID 一致** | `1` |
| `-Dsecret` | ✅ | 设备密钥（后台录入时指定） | 自定义 |
| `-DmikrotikHost` | ❌ | 内网路由器 IP | `192.168.1.1` |
| `-DmikrotikPort` | ❌ | 路由器 API 端口 | `8728` |
| `-DmikrotikUser` | ❌ | 路由器 API 用户名 | `hotelapi` |
| `-DmikrotikPassword` | ❌ | 路由器 API 密码 | 你的密码 |
| `-DlocalNetwork` | ❌ | 内网扫描范围 | `192.168.1.0/24` |

**一键部署为系统服务（推荐）：**

```bash
cd /opt/hotel-wifi/hotel-wifi-platform
bash mikrotik-scripts/deploy-agent.sh

# 编辑配置文件
vi /opt/hotel-connector/connector.env

# 启动并设置开机自启
systemctl enable hotel-connector
systemctl start hotel-connector
systemctl status hotel-connector

# 查看日志
journalctl -u hotel-connector -f
```

> 该脚本会自动创建：
> - `/opt/hotel-connector/` 工作目录
> - `start.sh` 启动脚本
> - `connector.env` 环境变量文件
> - `hotel-connector.service` systemd 服务（自动重启、崩溃恢复）

**阶段三：验证连接**

1. 回到管理后台 → 设备管理页面
2. 找到刚添加的设备，点击 **Agent** 按钮
3. 弹出提示 "Agent在线，命令可达" 即表示连接成功
4. 设备状态自动变为 🟢 **在线**
5. 再点击 **测试** 按钮验证路由器 REST API 可正常调用

### 11.3 添加设备表单详解

点击 **添加设备** 弹出以下表单：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| **设备名称** | 文本 | ✅ | 自定义设备标识，如 "XX酒店主路由"、"大堂AP" |
| **IP/域名** | 文本 | ✅ | **直连模式：** 路由器公网 IP 或域名<br>**Agent 模式：** 以 `ws://` 开头，如 `ws://agent-hotel-01`（系统据此自动判断模式） |
| **API 端口** | 数字 (1-65535) | ✅ | MikroTik REST API 端口，默认 **8728**。使用 HTTPS 时填 **443** |
| **API 用户名** | 文本 | ✅ | MikroTik 路由器登录用户名。为安全起见，建议在路由器上**专建 API 用户**而非使用 admin |
| **API 密码** | 密码 | ✅ | 路由器 API 用户的密码。编辑时留空表示不修改。**密码在数据库中使用 AES 加密存储** |
| **所属酒店ID** | 数字 | ✅ | 关联到哪个酒店。设备采集的流量和会话归属于该酒店 |
| **Hotspot Server** | 文本 | ❌ | MikroTik Hotspot 服务名称，如 `hotspot1`。用于系统自动操作 Hotspot 用户 |
| **IP 地址池** | 文本 | ❌ | 分配给上网用户的 IP 段，如 `10.10.0.0/24`。用于 NAT 和限速规则 |

> **安全建议：** 在 MikroTik 上创建专用的 API 用户，只赋予最小权限：
> ```
> /user add name=hotelapi password=强密码 group=api
> ```
> 或赋予 read+write+api 权限：
> ```
> /user group add name=hotel-api-group policy="read,write,api,rest-api"
> /user add name=hotelapi password=强密码 group=hotel-api-group
> ```

### 11.4 测试连接

1. 在设备列表中找到目标设备
2. 点击操作栏的 **测试** 按钮
3. 系统通过 REST API 调用路由器的 `/rest/system/resource` 端点

**结果判断：**

| 提示 | 含义 | 后续操作 |
|------|------|---------|
| 🟢 "连接成功！设备在线" | 网络可达 + 认证通过 | 无需处理 |
| 🔴 "连接失败，请检查配置" | 网络不通或认证失败 | 检查 IP/端口/防火墙/用户名密码 |

**常见失败原因：**

| 原因 | 排查方法 |
|------|---------|
| 路由器 IP 不可达 | 从能访问路由器的机器上 `ping` 该 IP |
| API 端口未开启 | 在路由器上执行 `/ip service print` 查看 `www-ssl` 状态 |
| 防火墙拦截 | 检查路由器防火墙规则和上层防火墙 |
| 用户名/密码错误 | 在路由器上重新确认 API 用户的密码 |
| Agent 未连接 | 先点击 **Agent** 按钮查看在线状态 |

### 11.6 设备状态说明

设备状态由后端根据心跳和连接测试结果自动更新：

| 状态 | 颜色 | 含义 | 触发条件 |
|------|------|------|---------|
| **在线** (ONLINE) | 🟢 绿色 | 设备正常 | Agent 心跳正常 或 直连测试通过 |
| **离线** (OFFLINE) | ⚪ 灰色 | 设备不可达 | 新建设备默认状态 / Agent 断线 |
| **异常** (ERROR) | 🔴 红色 | 通信异常 | 连接测试失败 |
| **维护** (MAINTENANCE) | 🟡 黄色 | 人工维护 | 管理员手动设置 |

**状态更新机制：**
- **Agent 模式：** 每 30 秒 Agent 发送心跳，服务器更新 `lastHeartbeat`，连续 90 秒无心跳则变更为离线
- **直连模式：** 仅在手动点击"测试"时更新状态
- **自动恢复：** Agent 重新连接后自动恢复在线状态

### 11.8 MikroTik 一键初始化脚本

在路由器上首次部署时，复制以下脚本到 **WinBox → New Terminal** 中执行：

```
/system script run setup-hotspot.rsc
```

脚本位于 `hotel-wifi-platform/mikrotik-scripts/setup-hotspot.rsc`，自动完成：

| 步骤 | 操作 | 说明 |
|:--:|------|------|
| 1 | 启用 REST API | 开启 `www-ssl` 服务（端口 443） |
| 2 | 创建 Hotspot Server | 配置 DHCP 地址池 + DNS + 空闲超时 |
| 3 | 配置 Walled Garden | 白名单放行 Portal 服务器和支付域名 |
| 4 | 设置 NAT 伪装 | 用户流量通过 WAN 口出互联网 |
| 5 | 创建 API 用户 | 专建 `hotelapi` 用户供系统调用 |
| 6 | 配置默认限速 | Simple Queue 规则：5M 上行 / 10M 下行 |
| 7 | 开启 SNMP | 用于设备监控（可选） |

> ⚠️ **执行前务必修改脚本中的变量：**
> - `serverIp` — 改为你的服务器公网 IP
> - `hotelName` — 改为酒店名称
> - `addressPool` — 改为实际的 IP 地址池
> - API 用户密码 — 改为强密码（默认 `ChangeMe123!`）

### 11.10 故障排查速查表

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 设备始终离线 | Agent 未启动或网络不通 | 检查 Agent 进程 `ps aux | grep connector`；查看日志 `journalctl -u hotel-connector -n 50` |
| 测试连接失败 | 路由器 API 未开启 / 防火墙拦截 | 在路由器执行 `/ip service print` 确认 `www-ssl` 已启用；检查防火墙规则 |
| Agent 频繁断线重连 | 网络不稳定 / 服务器不可达 | 检查内网到 `185.239.71.210:8080` 的网络连通性 |
| 无法扫描内网设备 | Agent 无法访问路由器 DHCP | 确认 `-DmikrotikHost` 和 `-DmikrotikUser` 参数正确 |
| 测试成功但流量不更新 | 流量采集服务未运行 | 检查服务器日志：`docker logs hotel-wifi-api --tail 100 | grep -i traffic`

## 13. 常见问题

### Q1: 忘记管理员密码怎么办？

连接数据库，执行以下 SQL 重置密码为 `admin123`：

```sql
UPDATE sys_user SET password = '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iKTVKIUi'
WHERE username = 'admin';
```

### Q2: 前端页面无法加载？

1. 确认后端 API 服务已启动：访问 `http://服务器IP:8080/doc.html`
2. 检查 Nginx 配置是否指向正确的前端和后端地址
3. 清除浏览器缓存后重试

### Q3: 会员反映无法上网？

按以下步骤排查：
1. **查状态**：会员管理 → 确认该会员状态为"正常"
2. **查余额**：确认余额充足或套餐未过期
3. **查设备**：设备管理 → 确认路由器状态为"在线"
4. **测试连接**：设备管理 → 对目标路由器点击"测试"

### Q4: 扣费不准确？

系统采用 30 秒轮询 MikroTik 获取流量数据，60 秒结算一次。扣费存在 1-2 分钟延迟属于正常现象。如需更高精度，可在 `application.yml` 中调整：

```yaml
billing:
  poll-interval-seconds: 15   # 缩小轮询间隔
  deduct-interval-seconds: 30 # 缩小扣费间隔
```

### Q5: 如何修改系统 Logo 和品牌名称？

前端品牌信息在以下文件中：
- `hotel-wifi-frontend/src/components/layout/AppLayout.vue` — 侧边栏 Logo
- `hotel-wifi-frontend/src/views/login/LoginView.vue` — 登录页标题
- `hotel-wifi-frontend/index.html` — 浏览器标签页标题

### Q6: 数据库备份建议

### Q7: RADIUS 设备配置要点？

1. 在 NAS 设备上配置 RADIUS 服务器地址为本系统 IP
2. 认证端口：**1812**，计费端口：**1813**
3. 共享密钥（Secret）需要在设备表 `router_device.api_password` 中配置
4. 设备类型选择 `HUAWEI` / `H3C` / `CISCO` / `JUNIPER`（非 MIKROTIK）
