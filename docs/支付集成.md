## 1. 概述

Monnify 是尼日利亚领先的支付网关，支持多种支付方式，提供完整的 API 体系用于酒店 WiFi 计费系统的在线支付。

### 核心特性
- **多支付方式**: 银行卡(CARD)、银行转账(ACCOUNT_TRANSFER)、USSD、手机号支付
- **储备账户**: 为每个会员生成专属虚拟银行账户，支持重复付款
- **Webhook 通知**: HMAC-SHA512 签名验证 + IP 白名单
- **子账户分账**: 支持最多 5 个子账户按比例或固定金额分账
- **退款**: 支持全额/部分退款，Webhook 异步通知结果

### 集成模式

| 模式 | 复杂度 | 适用场景 |
|------|:--:|------|
| 支付链接 (Payment Links) | 无代码 | 快速测试、无技术团队 |
| Web SDK (前端弹窗) | 低 | 住客 Portal 页面弹窗支付 |
| Mobile SDK | 中 | iOS/Android 原生 App |
| Direct API | 高 | 完全自定义支付流程（本项目采用） |

## 3. API 认证

### 3.1 获取 Bearer Token

Monnify 使用 **OAuth 2.0 Bearer Token** 认证。Token 有效期约 1 小时，需定时刷新。

**请求:**
```
POST /api/v1/auth/login
Host: https://sandbox.monnify.com  (测试)
Host: https://api.monnify.com       (生产)
Authorization: Basic base64(API_Key:Secret_Key)
```

**响应:**
```json
{
  "requestSuccessful": true,
  "responseMessage": "success",
  "responseCode": "0",
  "responseBody": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 3600
  }
}
```

**Java 实现（系统集成）：**
```java
@Component
public class MonnifyAuthService {
    private final OkHttpClient httpClient;
    private String accessToken;
    private long expiresAt;

    @Value("${payment.monnify.api-key}")
    private String apiKey;

    @Value("${payment.monnify.secret-key}")
    private String secretKey;

    @Value("${payment.monnify.base-url}")
    private String baseUrl; // https://sandbox.monnify.com 或 https://api.monnify.com

    public String getToken() {
        if (accessToken != null && System.currentTimeMillis() < expiresAt) {
            return accessToken;
        }
        return refreshToken();
    }

    private String refreshToken() {
        String auth = Base64.getEncoder()
            .encodeToString((apiKey + ":" + secretKey).getBytes());

        Request request = new Request.Builder()
            .url(baseUrl + "/api/v1/auth/login")
            .header("Authorization", "Basic " + auth)
            .post(RequestBody.create("", null))
            .build();

        try (Response response = httpClient.newCall(request).execute()) {
            String body = response.body().string();
            JsonNode node = new ObjectMapper().readTree(body);
            accessToken = node.get("responseBody").get("accessToken").asText();
            int expiresIn = node.get("responseBody").get("expiresIn").asInt();
            expiresAt = System.currentTimeMillis() + (expiresIn - 60) * 1000L; // 提前1分钟刷新
            return accessToken;
        } catch (IOException e) {
            throw new BusinessException(500, "Monnify认证失败: " + e.getMessage());
        }
    }
}
```

## 4. 支付初始化

### 4.1 初始化交易 (Initialize Transaction)

**请求:**
```
POST /api/v1/merchant/transactions/init-transaction
Host: https://sandbox.monnify.com 或 https://api.monnify.com
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**请求体:**
```json
{
  "amount": 100.00,
  "customerName": "住客姓名",
  "customerEmail": "guest@example.com",
  "paymentReference": "HW20260606001",
  "paymentDescription": "包月上网套餐 - ¥129",
  "currencyCode": "NGN",
  "contractCode": "2957251029",
  "redirectUrl": "https://你的域名/portal/success",
  "paymentMethods": ["CARD", "ACCOUNT_TRANSFER"],
  "metaData": {
    "memberId": "123",
    "packageId": "3",
    "orderNo": "HW20260606001"
  }
}
```

**参数说明:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `amount` | Number | ✅ | 支付金额 |
| `customerName` | String | ✅ | 客户姓名 |
| `customerEmail` | String | ✅ | 客户邮箱 |
| `paymentReference` | String | ✅ | **唯一**交易编号，重复会导致失败 |
| `paymentDescription` | String | 否 | 支付描述 |
| `currencyCode` | String | ✅ | 货币代码（NGN/USD等） |
| `contractCode` | String | ✅ | 合同编号 |
| `redirectUrl` | String | ✅ | 支付完成后跳转地址 |
| `paymentMethods` | Array | 否 | 限制支付方式 |
| `incomeSplitConfig` | Array | 否 | 分账配置（见§10） |
| `metaData` | Object | 否 | 自定义元数据（订单号/会员ID等） |

**响应:**
```json
{
  "requestSuccessful": true,
  "responseMessage": "success",
  "responseCode": "0",
  "responseBody": {
    "transactionReference": "MNFY|20260606001234|000001",
    "paymentReference": "HW20260606001",
    "amount": 100.00,
    "totalPayable": 101.50,
    "paymentMethod": "CARD",
    "paymentStatus": "PENDING",
    "checkoutUrl": "https://sandbox.sdk.monnify.com/checkout/MNFY|XXXX",
    "accountNumber": null,
    "accountName": null,
    "bankName": null,
    "ussdPayment": null
  }
}
```

**Java 实现（系统集成）：**
```java
@Service
public class MonnifyPaymentService {

    private final MonnifyAuthService authService;
    private final OkHttpClient httpClient;
    private final ObjectMapper mapper;
    private final BillingOrderMapper orderMapper;

    @Value("${payment.monnify.contract-code}")
    private String contractCode;

    @Value("${payment.monnify.base-url}")
    private String baseUrl;

    @Value("${payment.monnify.redirect-url}")
    private String redirectUrl;

    /**
     * 创建 Monnify 支付订单
     */
    public Map<String, Object> initTransaction(Long memberId, Long packageId,
                                                BigDecimal amount, String customerName,
                                                String customerEmail) {
        // 1. 生成本地订单号
        String paymentRef = "HW" + UUID.randomUUID().toString()
                .replace("-", "").substring(0, 12).toUpperCase();

        // 2. 存本地预支付订单
        BillingOrder order = new BillingOrder();
        order.setOrderNo(paymentRef);
        order.setTenantId(TenantContext.get());
        order.setMemberId(memberId);
        order.setPackageId(packageId);
        order.setAmount(amount);
        order.setPayType("MONNIFY");
        order.setStatus("PENDING");
        orderMapper.insert(order);

        // 3. 调 Monnify 初始化交易
        Map<String, Object> body = new HashMap<>();
        body.put("amount", amount.doubleValue());
        body.put("customerName", customerName);
        body.put("customerEmail", customerEmail);
        body.put("paymentReference", paymentRef);
        body.put("paymentDescription", "酒店WiFi套餐购买");
        body.put("currencyCode", "NGN");
        body.put("contractCode", contractCode);
        body.put("redirectUrl", redirectUrl);
        body.put("paymentMethods", Arrays.asList("CARD", "ACCOUNT_TRANSFER", "PHONE_NUMBER"));
        body.put("metaData", Map.of("memberId", memberId, "packageId", packageId));

        try {
            String json = mapper.writeValueAsString(body);
            Request request = new Request.Builder()
                .url(baseUrl + "/api/v1/merchant/transactions/init-transaction")
                .header("Authorization", "Bearer " + authService.getToken())
                .post(RequestBody.create(json, MediaType.parse("application/json")))
                .build();

            try (Response response = httpClient.newCall(request).execute()) {
                String respBody = response.body().string();
                JsonNode node = mapper.readTree(respBody);

                if (!node.get("requestSuccessful").asBoolean()) {
                    throw new BusinessException(400,
                        "支付初始化失败: " + node.get("responseMessage").asText());
                }

                JsonNode txBody = node.get("responseBody");

                Map<String, Object> result = new HashMap<>();
                result.put("orderNo", paymentRef);
                result.put("transactionReference", txBody.get("transactionReference").asText());
                result.put("checkoutUrl", txBody.get("checkoutUrl").asText());
                result.put("amount", amount);
                result.put("totalPayable", txBody.get("totalPayable").asDouble());

                return result;
            }
        } catch (IOException e) {
            throw new BusinessException(500, "Monnify支付请求失败: " + e.getMessage());
        }
    }
}
```

## 5. 支付方式

### 5.1 支付方式枚举

| 代码 | 说明 | 用户体验 |
|------|------|---------|
| `CARD` | 银行卡支付（Visa/Mastercard/Verver） | Web SDK 弹窗输入卡号 |
| `ACCOUNT_TRANSFER` | 银行转账（通过储备账户） | 生成虚拟账号，用户转账 |
| `PHONE_NUMBER` | 手机号支付（USSD 无银行卡） | 手机拨号 USSD 码支付 |

### 5.2 储备账户支付流程

```
1. 系统调用 Create Reserved Account API
2. Monnify 返回虚拟银行账号
3. 住客通过银行 App 向该账号转账
4. Monnify 收到转账后通过 Webhook 通知系统
5. 系统更新订单状态 → 激活会员套餐
```

### 5.3 支付页面跳转

初始化交易成功后，将住客重定向到 `checkoutUrl`，或使用 Web SDK 在当前页面弹窗。

## 6. Webhook 回调处理

### 6.1 签名验证算法

Monnify 使用 **HMAC-SHA512** 对回调请求体进行签名，签名放在 HTTP Header 的 `monnify-signature` 字段。

**计算公式:**
```
monnify-signature = HMAC-SHA512(Secret_Key, requestBody)
```

**Java 实现:**
```java
/**
 * 验证 Monnify Webhook 签名
 *
 * @param requestBody  原始请求体（保持原样，不修改空格/格式）
 * @param signature    请求头中的 monnify-signature
 * @param secretKey    Monnify Secret Key
 * @return true=签名有效
 */
public static boolean verifySignature(String requestBody, String signature, String secretKey) {
    try {
        Mac mac = Mac.getInstance("HmacSHA512");
        SecretKeySpec keySpec = new SecretKeySpec(secretKey.getBytes(StandardCharsets.UTF_8), "HmacSHA512");
        mac.init(keySpec);
        byte[] hashBytes = mac.doFinal(requestBody.getBytes(StandardCharsets.UTF_8));

        // 转为小写十六进制字符串
        StringBuilder hexString = new StringBuilder();
        for (byte b : hashBytes) {
            hexString.append(String.format("%02x", b));
        }

        // 安全比较（防时序攻击）
        return MessageDigest.isEqual(
            hexString.toString().getBytes(StandardCharsets.UTF_8),
            signature.getBytes(StandardCharsets.UTF_8)
        );
    } catch (Exception e) {
        log.error("Monnify签名验证异常", e);
        return false;
    }
}
```

### 6.2 Webhook 处理流程

```
Monnify → POST /api/v1/payment/monnify/notify
    │
    ├─ 1. 验证来源 IP 是否为 35.242.133.146
    ├─ 2. 验证 monnify-signature 签名
    ├─ 3. 解析请求体，提取 paymentReference
    ├─ 4. 检查本地订单是否存在，防止重复处理
    ├─ 5. 查询 Monnify 交易状态（二次确认，防止伪造）
    ├─ 6. 更新订单状态（PAID）→ 充值会员余额/激活套餐
    └─ 7. 返回 HTTP 200 OK
```

### 6.4 Controller 实现

```java
@RestController
@RequestMapping("/api/v1/payment/monnify")
public class MonnifyWebhookController {

    private final MonnifyPaymentService paymentService;

    @Value("${payment.monnify.secret-key}")
    private String secretKey;

    /** Monnify 支付回调（不走JWT认证） */
    @PostMapping("/notify")
    public ResponseEntity<String> handleNotification(
            @RequestBody String payload,
            @RequestHeader("monnify-signature") String signature,
            HttpServletRequest request) {

        // 1. IP 白名单校验
        String remoteIp = request.getRemoteAddr();
        if (!"35.242.133.146".equals(remoteIp)) {
            log.warn("Monnify回调IP不匹配: {}", remoteIp);
            return ResponseEntity.status(403).body("Forbidden");
        }

        // 2. 签名验证
        if (!MonnifySignatureUtil.verifySignature(payload, signature, secretKey)) {
            log.error("Monnify回调签名验证失败");
            return ResponseEntity.status(401).body("Invalid signature");
        }

        // 3. 处理业务逻辑
        try {
            paymentService.handleWebhook(payload);
            return ResponseEntity.ok("OK");
        } catch (Exception e) {
            log.error("Monnify回调处理异常", e);
            return ResponseEntity.status(500).body("Error");
        }
    }
}
```

## 7. 交易状态查询

### 7.1 查询交易

```
GET /api/v2/transactions/{transactionReference}
Host: https://sandbox.monnify.com 或 https://api.monnify.com
Authorization: Bearer {accessToken}
```

**Java 实现:**
```java
public Map<String, Object> verifyTransaction(String transactionReference) {
    Request request = new Request.Builder()
        .url(baseUrl + "/api/v2/transactions/" + URLEncoder.encode(transactionReference, "UTF-8"))
        .header("Authorization", "Bearer " + authService.getToken())
        .get()
        .build();

    try (Response response = httpClient.newCall(request).execute()) {
        JsonNode node = mapper.readTree(response.body().string());
        JsonNode body = node.get("responseBody");
        return Map.of(
            "status", body.get("paymentStatus").asText(),
            "amount", body.get("amountPaid").asDouble(),
            "reference", body.get("paymentReference").asText()
        );
    }
}
```

## 8. 储备账户 Reserved Account

### 8.1 创建储备账户

为每个会员创建专属虚拟银行账户，用户可通过银行转账自动关联。

```
POST /api/v1/bank-transfer/reserved-accounts
Authorization: Bearer {accessToken}
```

**请求体:**
```json
{
  "accountReference": "HW_MEMBER_123",
  "accountName": "ALEN WIFI / 张三",
  "currencyCode": "NGN",
  "contractCode": "2957251029",
  "customerEmail": "guest@example.com",
  "customerName": "张三",
  "bvn": "12345678901",
  "getAllAvailableBanks": true,
  "incomeSplitConfig": [],
  "restrictPaymentSource": false
}
```

**响应:**
```json
{
  "responseBody": {
    "accountNumber": "0123456789",
    "accountName": "ALEN WIFI / 张三",
    "bankName": "Moniepoint MFBank",
    "bankCode": "50515",
    "reservationReference": "RES_REF_001",
    "accountReference": "HW_MEMBER_123",
    "status": "ACTIVE"
  }
}
```

### 8.2 限制和注意事项

- 必须提供 **BVN** 或 **NIN**（尼日利亚银行认证号）
- 同一 `accountReference` 不可重复使用
- 同一 `customerEmail` 只能有一个活跃的储备账户
- 可设置 `limitProfileCode` 限制单笔/单日金额

## 10. 子账户与分账

### 10.1 使用场景
- 酒店管理集团抽取佣金
- 多个分店共享收入
- 平台服务费

### 10.2 配置方式

需先联系 `integration-support@monnify.com` 激活分账功能，然后在支付请求中添加：

```json
"incomeSplitConfig": [
  {
    "subAccountCode": "MFY_SUB_319452883228",
    "feePercentage": 10.5,
    "splitPercentage": 20,
    "feeBearer": true
  }
]
```

| 字段 | 说明 |
|------|------|
| `subAccountCode` | 子账户代码 |
| `feePercentage` | 手续费承担比例 (0-100) |
| `splitPercentage` | 分账比例 (0-100) |
| `splitAmount` | 固定分账金额（与 splitPercentage 二选一） |
| `feeBearer` | 是否承担交易手续费 |

## 13. 错误码

### 13.1 HTTP 状态码

| HTTP | 含义 |
|------|------|
| 200 | 请求成功 |
| 400 | 参数错误或业务逻辑拒绝 |
| 401 | 认证失败（Token 过期/无效） |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | Monnify 服务器内部错误 |

### 13.2 业务错误信息

| 错误信息 | 原因 | 解决 |
|------|------|------|
| "Reusing the same paymentReference" | 重复的交易编号 | 每次生成新的 UUID |
| "Invalid contract code" | Contract Code 错误 | 从后台 Settings → Contract Setup 获取 |
| "BVN or NIN is required" | 缺少身份认证 | 提供 BVN 或 NIN |
| "Unknown currency code" | 不支持的货币 | 确认货币代码（NGN/USD） |
| "You can not reserve two accounts with the same reference" | accountReference 重复 | 使用唯一标识 |
| "Unknown sub account code" | 子账户不属于当前商户 | 确认 subAccount 正确 |
| "We do not support virtual accounts from banks with code XXX" | 不支持的银行代码 | 重新请求支持的银行 |
| "R42" - 重复客户邮箱 | 同一 email 已有储备账户 | 使用唯一邮箱或共享已创建账户 |

### 13.3 响应统一格式

```json
{
  "requestSuccessful": true,     // true/false
  "responseMessage": "success",  // 描述信息
  "responseCode": "0",           // "0"=成功, 其他=见错误码
  "responseBody": { ... }        // 业务数据
}
```

## 14. 系统对接实现

### 14.1 配置文件 (`application-production.yml`)

```yaml
payment:
  monnify:
    base-url: https://api.monnify.com      # 生产; 测试用 https://sandbox.monnify.com
    api-key: ${MONNIFY_API_KEY}
    secret-key: ${MONNIFY_SECRET_KEY}
    contract-code: ${MONNIFY_CONTRACT_CODE}
    redirect-url: https://你的域名/portal/success
    webhook-ip: 35.242.133.146
```

### 14.2 系统支付流程图

```
住客 Portal
    │
    ├─ 选择套餐 → 点击支付
    │
    ▼
POST /api/v1/payment/monnify/create
    │ 后端: MonnifyPaymentService.initTransaction()
    │ 1. 生成本地订单 (status=PENDING)
    │ 2. 调 Monnify Init Transaction API
    │
    ▼
返回 checkoutUrl → 住客完成支付
    │
    ├─ 方式A: Web SDK 弹窗 → 输入卡号
    ├─ 方式B: 跳转 checkoutUrl
    └─ 方式C: 银行转账(储备账户)
           │
           ▼
Monnify 异步回调 POST /api/v1/payment/monnify/notify
    │
    ├─ IP 白名单验证
    ├─ HMAC-SHA512 签名验证
    ├─ 解析 paymentReference
    ├─ 查询 Monnify 确认交易状态
    ├─ 更新本地订单(PAID)
    ├─ 充值会员余额 + 激活套餐到期时间
    └─ 返回 200 OK
```

### 14.3 安全注意事项

1. **Token 缓存**: 在 Redis 中缓存 accessToken，减少 API 调用频率
2. **签名验证**: **绝对不要跳过** Webhook 签名验证
3. **幂等性**: 使用 `paymentReference` 防止重复处理同一笔支付
4. **金额校验**: Webhook 处理时比较 `amountPaid` 与本地订单金额
5. **二次确认**: 收到 Webhook 后主动查询 Monnify 确认交易状态
6. **超时处理**: Monnify 交易有超时时间（通常 30 分钟），超时自动取消
